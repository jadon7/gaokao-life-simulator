# 高考人生模拟器协议 vNext

这份协议用于统一三件事：

1. DeepSeek 的结构化输出格式
2. 前端卡片 UI 的展示字数预算
3. 霍兰德兴趣侧写的隐藏计分规则

目标不是让一次模型输出包办所有职责，而是把系统拆成三层：

- 故事层：负责 18 张牌的连续剧情、反转、伏笔和双线推进
- 展示层：负责把内容压进固定 UI，不爆框，不难读
- 测量层：负责 RIASEC 隐藏计分，不再从表层文案反推

## 一、产品定位

这一版的产品定位改为：

**18 张牌的连续人生短剧 + 霍兰德兴趣倾向侧写**

这意味着：

- 18 张牌的第一职责是讲一个有起伏、有回收、有正向落点的故事
- 霍兰德结果是从 18 次选择里观察到的兴趣倾向，不是严格心理测评
- 结果页文案必须避免“你天生就是 XX”这种判定口吻
- 结果页更适合写成“你这局更像哪几类人”

## 二、三层模型

### 1. 故事层 StoryState

故事层负责：

- 当前第几年
- 当前处于哪一幕
- 本年主线属于生活线还是关系线
- 生活线当前状态
- 关系线当前状态
- 本年场景冲突
- 可回响种子
- A/B 两个方向的真实后果倾向
- 每个选项的隐藏 RIASEC 分值

故事层输出给系统，不直接照搬到 UI。

### 2. 展示层 CardView

展示层负责：

- 把故事状态翻译成手机端可读卡片
- 控制标题、正文、便利贴、选项字数
- 负责好读、好选、好截图

展示层只消费 StoryState 的一部分字段。

### 3. 测量层 HollandScore

测量层负责：

- 读取每个选项背后的隐藏 `riasec`
- 聚合 18 次选择结果
- 输出 RIASEC 图表和解释

测量层不再从以下内容反推：

- 选项表层措辞
- 左右位置
- scene 文案关键词

## 三、年度卡牌协议

## 3.1 StoryStateCard

每一张牌的模型主输出结构：

```json
{
  "year": 7,
  "phase": "第一次付代价",
  "mainTrack": "life",
  "lifeTrack": "你开始同时扛专业成绩和外部项目，睡眠被切碎了。",
  "relationshipTrack": "你和沈枝聊天变频繁，但一直没把话说透。",
  "summary": "外部项目让你在学院里有了名字，代价是作息彻底乱掉；沈枝开始记得你每次嘴硬后的沉默。",
  "scene": {
    "title": "答应的事撞车了",
    "body": "学院展示周临近，导师临时把最难的部分压给你，周越那边的小项目也卡在交付前夜。两边都在等你，手机像同时开了两个倒计时。"
  },
  "a": {
    "title": "先保学院",
    "desc": "把公开场合这一战先打稳",
    "tag": "守正面",
    "riasec": { "R": 0, "I": 1, "A": 0, "S": 0, "E": 0, "C": 3 }
  },
  "b": {
    "title": "先救项目",
    "desc": "赌一次外部机会的后劲",
    "tag": "抢窗口",
    "riasec": { "R": 1, "I": 0, "A": 0, "S": 0, "E": 3, "C": 0 }
  },
  "callbacks": ["周越的小项目", "沈枝", "展示周翻车预警"]
}
```

## 3.2 字段定义

- `year`
  - 当前年份，1-18
- `phase`
  - 当前阶段名，只给系统和调试用
- `mainTrack`
  - 本年正式问题的主线
  - 只允许 `life` 或 `relationship`
- `lifeTrack`
  - 生活线当前状态
  - 包含学业、事业、家庭、朋友、健康、财务、居住等
- `relationshipTrack`
  - 关系线当前状态
  - 包含好感、暧昧、恋爱、错过、陪伴、关系支持系统
- `summary`
  - 卡片外便利贴内容
  - 写“上一年后果 + 本年另一条线的同步近况”
- `scene.title`
  - 卡片标题
- `scene.body`
  - 卡片正文
- `a` / `b`
  - 两个可立即选择的方向
- `a.riasec` / `b.riasec`
  - 隐藏计分字段，不展示给用户
- `callbacks`
  - 后面可回收的角色、道具、梗或未完事件

## 四、展示层字数预算

所有 UI 文案必须以展示预算为先，不允许先长写再裁剪。

### 4.1 卡片区

- `summary`
  - 26-40 个汉字
  - 1-2 句
  - 便利贴样式展示
  - 必须优先写清“谁、做了什么、关系或局面怎么变”
  - 如果涉及已出场关键角色，优先直接写名字，不要用“有人”“某个人”代替
  - 如果这一年没有太多新信息，宁可更短，也不要拿空话凑字数

- `scene.title`
  - 10-16 个汉字
  - 必须像一句完整的小标题

- `scene.body`
  - 72-108 个汉字
  - 2-4 句短句
  - 必须有场景、人物、冲突
  - 每一句都要尽量提供新信息，不允许抒情句占位置

### 4.2 选项区

- `a` / `b` 整体展示文案
  - 22-30 个汉字
  - 必须一眼能看懂“你到底准备怎么做”
  - 不能只是态度词，必须带处理方式
  - 结构建议：
    - 前半句：动作决策
    - 后半句：代价、优先级或判断依据
  - 例如：
    - `先把比赛机会抢下来，挂科风险后面再补救`
    - `先把绩点和作息守住，不让大学开局直接失控`

- `a.title` / `b.title`
  - 4-8 个汉字
  - 负责给按钮一个清楚的动作抓手
  - 例如：`先抢机会`、`先保底盘`

- `a.desc` / `b.desc`
  - 16-24 个汉字
  - 必须补足这一步的真实信息量
  - 不能只写空泛态度，至少要包含以下三者中的两项：
    - 你优先保什么
    - 你愿意冒什么风险
    - 你判断局势的依据是什么

- `a.tag` / `b.tag`
  - 2-4 个汉字
  - 只写选择气质
  - 例如：`抢窗口`、`守底盘`、`先回应`、`先止损`

### 4.2.1 选项信息量规则

选项加字数，不是为了更啰嗦，而是为了让系统更容易判断玩家的稳定偏好。

每个 A/B 至少要暴露下面 4 层信息中的 3 层：

1. **优先级**
   - 你先保什么
   - 例如：成绩、机会、关系、体面、情绪、现金流

2. **行动方式**
   - 你准备怎么做
   - 例如：硬冲、复盘、补救、解释、拖延、止损、协商、公开

3. **风险态度**
   - 你愿不愿意承担不确定性
   - 例如：先赌一把、先稳一手、先观察、先留后路

4. **判断视角**
   - 你是从什么角度在做决定
   - 例如：从人出发、从结果出发、从规则出发、从表达出发、从资源出发

只有这样，隐藏 `riasec` 才不会只是作者手填标签，而是能和可见文案互相印证。

### 4.2.2 选项和 RIASEC 的对应原则

为了让霍兰德结果变化更大、更稳，选项文案本身也要承担可识别信号。

- `R`
  - 文案里更常出现：动手、现场、自己顶上、立刻补、做出来、扛住
- `I`
  - 文案里更常出现：分析、验证、拆开看、先弄清原因、复盘、判断依据
- `A`
  - 文案里更常出现：表达、讲法、呈现、风格、内容、创意、作品感
- `S`
  - 文案里更常出现：回应、解释、安抚、陪、接住、顾人感受、先把话说开
- `E`
  - 文案里更常出现：争取、拍板、抢机会、带头、拉资源、推进、上桌
- `C`
  - 文案里更常出现：保底、流程、边界、计划、稳住、先守、按规则

注意：

- 不是要求机械堆关键词
- 而是要求每个选项的思维方式足够明确
- 前台文案和后台 `riasec` 要说同一种人话

### 4.2.3 杜绝空话和机械推进句

所有卡片文案都要尽量做到：

- 每一句都推进剧情
- 每一句都最好能回答“谁、在哪、做了什么、造成什么变化”
- 如果一句话回答不了这些，通常就是信息密度不够

以下属于要尽量杜绝的空话：

- “有人开始稳定关心你的饭点和心情”
- “你把这件事往前推进了一步”
- “新的机会和压力一起冒了出来”
- “局面变了，旁边的人和节奏也跟着动了”
- “留下了后果”
- “关系和生活都往前走了一步”

这些句子的问题不是文风，而是：

- 没写清是谁
- 没写清发生了什么
- 没写清变化落在哪里
- 占了字数，却没有给玩家新的剧情信息

更好的写法应该是：

- “沈枝连续三天把晚饭照片发给你，问你是不是又拿咖啡顶一顿”
- “周越先把你名字报了上去，导师第二天就把展示最难的那段塞给你”
- “合作方把试用名额压到今晚确认，你第一次发现这事真不是玩票”

原则只有一个：

**如果没有新信息，就缩短；不要用文学腔或总结腔把空白撑满。**

### 4.2.4 行文气质要求

整体语气要：

- 轻松
- 清楚
- 有一点幽默
- 像现在最容易读下去的热门小说或热帖

不要写成：

- 文学散文
- 高考阅读理解
- 长句堆叠的抒情作品
- 过度端着的青春疼痛文学

更推荐的感觉是：

- 短句
- 具体
- 带一点吐槽感
- 能让人顺着往下翻

允许有一点俏皮，但不要为了俏皮牺牲信息量。

### 4.3 结果页区

- 结果大标题
  - 10-18 个汉字
  - 直接输出内容，不输出空标题词

- `status42`
  - 60-90 个汉字

- `majorCareerNote`
  - 80-130 个汉字
  - 含 3 个概率结果 + 1 句总结

- `famousScenes`
  - 3 个块
  - 每块标题 10-18 个汉字
  - 每块正文 22-40 个汉字

- `trajectory`
  - 120-180 个汉字

- `shareHooks`
  - 结果页 UI 默认不展示
  - 仅保留给分享、文案或评论区扩展用途

## 五、18 张牌叙事节奏

18 张牌不是 18 个独立问题，而是 18 集连续短剧。

## 5.1 六段结构

### 第一段：开局被动入场（1-3）

目标：

- 快速建立身份
- 给出 2-3 个现实麻烦
- 同时种下一个关系线种子

要求：

- 不要开局太顺
- 第 2-3 年必须出现关系线的第一次轻微波动

### 第二段：第一次上头（4-6）

目标：

- 玩家开始觉得某条路能走通
- 生活线出现第一次快进
- 关系线开始有更明确的存在感

要求：

- 至少 1 个“以为自己赚到了”的节点
- 至少 1 个轻喜剧或社死型包袱

### 第三段：第一次付代价（7-9）

目标：

- 前面选择开始反噬
- 便利贴必须明显承载后果
- 关系线或生活线其中一条被迫让位

要求：

- 至少 1 次“承诺撞车”
- 至少 1 次“代价比预想更真实”

### 第四段：关系或生活翻面（10-12）

目标：

- 双线换镜头
- 前面没当主线的一条，这时要顶上来
- 节奏更像短剧中段反转

要求：

- 不能继续平推
- 必须出现“之前理解错了”或“之前太天真了”

### 第五段：底牌暴露（13-15）

目标：

- 早年的人、事、道具、承诺回来
- 进入回收前的高压段

要求：

- 至少 2 个 callback 回收
- 至少 1 个关键人物立场变化

### 第六段：最终回收与正向落点（16-18）

目标：

- 回收主线
- 给出阶段性答案
- 结尾允许有遗憾，但整体必须正向

要求：

- 不写成鸡汤
- 不写成命运判决书
- 要让人想截图

## 六、双线推进规则

## 6.1 线的定义

- `lifeTrack`
  - 学业、职业、家庭、朋友、健康、财务、城市、居住

- `relationshipTrack`
  - 好感、暧昧、恋爱、分开、错过、搭子、陪伴、支持系统

## 6.2 主副线规则

每一年：

- `scene` 只拍一条主线
- `summary` 只拍另一条线
- `lifeTrack` 和 `relationshipTrack` 两条都必须更新

## 6.3 交错规则

- 同一类型内容不能连续 3 年都占主线
- 恋爱/亲密关系不能连续多年都做 `scene`
- 生活线也不能把关系线长期挤没

## 6.4 便利贴规则

`summary` 只做两件事：

1. 写上一年选择已经造成的后果
2. 带一条本年未占主线的同步近况

禁止：

- 复述 scene
- 写成“近况播报”
- 写成抽象总结
- 写“你选择了 A”

## 七、出题规则

## 7.1 题目职责

每道题必须同时满足：

- 剧情上能推动故事
- 选择上能立刻做决定
- 测量上能提供至少一个 RIASEC 倾向信号

## 7.2 选项设计

选项设计优先级：

1. 方向明确
2. 代价不同
3. 气质不同
4. 对应不同隐藏 RIASEC

避免：

- 两个选项都只是“帮，但换一种帮法”
- 两个选项只是态度轻重不同
- 两个选项都像温和协商

## 7.3 题型池

18 张里建议覆盖以下题型：

- 资源争取题
- 风险处理题
- 承诺冲突题
- 公开表达题
- 人际边界题
- 创意表达题
- 分析判断题
- 动手执行题
- 家庭拉扯题
- 错过补救题
- callback 回收题

## 八、霍兰德隐藏计分协议

## 8.1 定位

霍兰德是“兴趣倾向侧写”，不是严格心理测验。

结果页表述必须像：

- “你这局更偏向……”
- “你的选择反复显示出……”

不要写成：

- “你天生就是……”
- “你一定适合……”

## 8.2 计分方式

每个选项自带 `riasec`：

```json
{
  "R": 0,
  "I": 1,
  "A": 0,
  "S": 0,
  "E": 0,
  "C": 3
}
```

规则：

- 单选项建议总权重在 2-5 之间
- 1 个选项可以同时命中 1-2 个维度
- 不建议 1 个选项同时命中 3 个以上维度

## 8.3 六类信号定义

- `R`
  - 动手、执行、实操、现场解决、真实物理世界

- `I`
  - 分析、研究、验证、观察、逻辑拆解

- `A`
  - 表达、创意、设计、内容、风格、讲述

- `S`
  - 倾听、帮助、支持、照顾、协调、接住别人

- `E`
  - 推动、争取、说服、组织、带头、调资源

- `C`
  - 计划、规则、秩序、稳定、校对、流程、风险控制

## 8.4 题型与计分映射建议

- 资源争取题
  - 重点区分 `E / C`

- 分析判断题
  - 重点区分 `I / E` 或 `I / C`

- 创意表达题
  - 重点区分 `A / C` 或 `A / E`

- 人际支持题
  - 重点区分 `S / C` 或 `S / E`

- 动手执行题
  - 重点区分 `R / I` 或 `R / C`

## 8.5 结果生成规则

- 按 18 次选择累计总分
- 取前三高作为三字母代码
- 百分比建议按总分占比做归一，不按最高值强制拉满

推荐：

- 用总分归一，避免每局都出现一个 100%
- 保留小数或四舍五入后控制差距真实感

## 十、明确删除的旧思路
